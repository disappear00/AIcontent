## 一、定义与理论框架

### 1.1 PEAS 模型与感知模块定位

在人工智能经典理论中，**PEAS 模型**（Performance, Environment, Actuators, Sensors）是描述智能体（Intelligent Agent）任务环境的标准化框架。其中 **Sensors（传感器）** 正是感知模块的理论原型——它定义了智能体如何从外部世界获取信息。

```
┌─────────────────────────────────────────────────────┐
│                    PEAS 模型                          │
├───────────────┬──────────────┬───────────┬──────────┤
│  Performance  │  Environment │ Actuators │ Sensors  │
│   性能度量     │    环境      │   执行器   │  传感器  │
│               │              │           │          │
│  "做得好不好" │  "在什么世界" │ "怎么影响" │ "怎么感知"│
└───────────────┴──────────────┴───────────┴──────────┘
                            ↑
                    感知模块的理论根基
```

在 AI Agent 语境下，**Sensors 被具象化为感知模块（Perception Module）**，其核心职责包括：

| 职责 | 描述 |
|------|------|
| **信号采集** | 从 API、网页、文件系统、用户输入、传感器读取原始数据 |
| **信号转换** | 将异构的原始数据（JSON、HTML、图像、音频）转化为统一的结构化表示 |
| **上下文注入** | 将转换后的 Observations 注入 LLM 的上下文窗口 |
| **记忆更新** | 将关键感知结果持久化到向量数据库或关系数据库 |

**核心定义：** 感知模块 = `Sensors(PEAS)` ∪ `Signal Transformer` ∪ `Context Builder`

---

### 1.2 Agent Loop 中的感知环节

**Agent Loop（智能体循环）** 的经典范式为：

```
Observe → Think → Act → Observe → Think → Act → ...
```

这是一个永不终止的反馈闭环。感知模块正是这个闭环的**入口**和**反馈接收器**：

```
┌──────────────────────────────────────────────┐
│                 Agent Loop                    │
│                                              │
│   ┌──────────┐    ┌──────────┐              │
│   │ OBSERVE  │───→│  THINK   │              │
│   │ (感知)   │    │ (思考)   │              │
│   └──────────┘    └────┬─────┘              │
│        ↑               │                    │
│        │         ┌─────▼─────┐              │
│   ┌────┴─────┐  │    ACT    │              │
│   │ Environment│ │  (行动)   │              │
│   │   环境     │  └─────┬─────┘              │
│   └───────────┘        │                    │
│        ↑               │                    │
│        └───────────────┘                    │
│         工具调用返回结果                      │
└──────────────────────────────────────────────┘
```

**感知模块在 Loop 中的两个关键触发点：**

1. **初始感知（Initial Observation）：** 用户输入目标后，Agent 从环境收集初始状态信息（如文件列表、API 状态、网页内容）。
2. **反馈感知（Feedback Observation）：** 每次 Action 执行后，工具返回的结果（如 HTTP Response、SQL 查询结果、Shell 输出）被感知模块捕获并转换为下一轮 Thought 的输入。

### 1.3 从原始信号到 Observation 的信号转化链

感知模块的核心技术挑战在于**异构信号的统一转化**。以下是一个完整的信号转化链（Signal Transformation Pipeline）：

```
原始信号 (Raw Signal)
    │
    ▼
┌─────────────────┐
│  Signal Receiver │  ← 多协议采集层 (HTTP/WS/gRPC/File I/O)
│  信号接收器       │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  Type Detector   │  ← 自动识别信号类型 (JSON/HTML/Image/Audio/Video)
│  类型检测器       │
└────────┬────────┘
         │
    ┌────┴────┬──────────┬──────────┐
    ▼         ▼          ▼          ▼
┌──────┐ ┌──────┐ ┌────────┐ ┌────────┐
│ JSON │ │HTML  │ │ Image  │ │ Audio  │
│Parser│ │Parser│ │Encoder │ │Encoder │
└──┬───┘ └──┬───┘ └───┬────┘ └───┬────┘
   │        │         │          │
   └────────┴────┬────┴──────────┘
                 ▼
┌─────────────────────────────┐
│  Observation Normalizer     │  ← 统一为 LLM 可理解的文本/多模态令牌
│  Observation 标准化器        │
└────────────┬────────────────┘
             │
             ▼
┌─────────────────────────────┐
│  Context Assembler          │  ← 组装为 System/User/Assistant Message
│  上下文组装器                │
└─────────────────────────────┘
```

**关键设计原则：**

- **类型无损转换：** 结构化数据（JSON）应保留其层次关系，HTML 应保留 DOM 语义结构。
- **信息压缩策略：** 面对超大响应体（如 100KB+ 的 API 返回），需要智能截断或摘要化，避免撑爆 LLM 上下文窗口。
- **时间戳标记：** 每条 Observation 应打上时间戳和来源标签，以支持 Trace 追踪。

## 二、技术演进：从单模态到全域感知

### 2.1 2024 年：纯文本上下文注入时代

2024 年的 AI Agent 感知能力栈呈现出显著的**单模态局限**：

```
┌─────────────────────────────────────────┐
│          2024 感知能力栈                  │
│                                         │
│  ┌─────────────────────────────────┐   │
│  │       Text Injection Layer       │   │
│  │  • JSON 序列化为文本             │   │
│  │  • HTML 提取纯文本              │   │
│  │  • CSV/TXT 直接嵌入              │   │
│  │  • API Response → str()          │   │
│  └─────────────────────────────────┘   │
│                                         │
│  核心局限：                              │
│  ✗ 无法处理图像/视频/音频               │
│  ✗ 结构化信息在序列化中丢失              │
│  ✗ 上下文窗口有限（4K-128K tokens）     │
│  ✗ 缺乏统一的工具接入标准                │
└─────────────────────────────────────────┘
```

**典型实现模式（以 2024 年初的 AutoGPT 为例）：**

```python
# AutoGPT Classic - 典型的文本感知模式
class CommandRegistry:
    """所有工具返回结果都以纯文本形式注入 Prompt"""
    
    def execute(self, command_name: str, arguments: dict) -> str:
        result = self.commands[command_name](**arguments)
        # 关键步骤：所有结果强制转为文本
        return f"Command {command_name} returned:\n{str(result)}"
```

这一阶段的感知模块本质上是 **"Stringify Everything"** 模式，信息保真度受限于文本的线性表达能力。

**核心感知代码路径：**

```python
# AutoGPT Classic - Agent Protocol 感知循环
class Agent:
    """AutoGPT 的核心 Agent 类，实现完整的感知-思考-行动闭环"""
    
    def __init__(self, memory_backend="pinecone"):
        self.memory = MemoryBackend(backend=memory_backend)
        self.commands = CommandRegistry()
    
    def step(self) -> dict:
        # 1. 感知阶段：从记忆系统检索相关上下文
        relevant_memories = self.memory.retrieve_relevant(
            query=self.current_task,
            top_k=5
        )
        
        # 2. 构建包含历史观察的 Prompt
        prompt = self._build_prompt(
            goal=self.goal,
            memories=relevant_memories,
            # 关键：上一轮的工具执行结果作为观察注入
            last_observation=self.history[-1] if self.history else None
        )
        
        # 3. LLM 思考 + 行动
        response = self.llm.generate(prompt)
        action = self._parse_action(response)
        
        # 4. 执行行动并获取新的观察
        observation = self.commands.execute(
            command=action["name"],
            arguments=action["args"]
        )
        
        # 5. 将观察存入记忆
        self.memory.store(
            content=observation,
            metadata={"action": action["name"], "timestamp": now()}
        )
        
        return {"thought": response, "observation": observation}
```

**插件感知模式：**

```yaml
# plugins_config.yaml - AutoGPT 的插件感知配置
plugins:
  - id: web_search
    enabled: true
    config:
      engine: google
      max_results: 5
  - id: file_operations
    enabled: true
    config:
      allowed_paths: ["/workspace"]
  - id: code_execution
    enabled: true
    config:
      sandbox: docker
      timeout: 30s
```

**设计要点：**

- **插件即传感器：** 每个插件本质上是一个 Sensor 实例，负责采集特定类型的环境信号。
- **记忆分层：** 短期记忆（当前会话）存储在 JSON 文件中，长期记忆通过 Pinecone/ChromaDB 向量化存储。
- **Observation 聚合：** 多个插件的返回结果通过 `Observation Aggregator` 统一去重、排序、截断后再注入 Prompt。

#### BabyAGI：向量记忆驱动的任务感知

**项目地址：** `yoheinakajima/babyagi`（GitHub）

BabyAGI 展示了如何通过**向量数据库**实现持续的环境感知与任务追踪。其核心创新在于使用**向量检索**替代简单的历史记录追加。

**BabyAGI 感知循环：**

```python
# BabyAGI - 感知与记忆管理的核心实现
import openai
import pinecone
from chromadb import Client as ChromaClient

class BabyAGI:
    def __init__(
        self,
        vector_store="chroma",  # 支持 Pinecone / ChromaDB / Qdrant
        embedding_model="text-embedding-3-small"
    ):
        if vector_store == "chroma":
            self.vector_db = ChromaClient()
            self.collection = self.vector_db.create_collection("babyagi_memory")
        elif vector_store == "pinecone":
            self.vector_db = pinecone.Index("babyagi")
        
        self.embedding_model = embedding_model
        self.task_list = []
    
    def perceive_context(self, task: dict, top_k: int = 5) -> str:
        """
        感知模块核心：从向量数据库检索与当前任务最相关的历史观察
        """
        # 1. 将当前任务向量化
        task_embedding = openai.embeddings.create(
            model=self.embedding_model,
            input=task["description"]
        ).data[0].embedding
        
        # 2. 语义相似度检索
        results = self.collection.query(
            query_embeddings=[task_embedding],
            n_results=top_k
        )
        
        # 3. 将检索到的记忆拼装为上下文 Observation
        context = "Relevant past observations:\n"
        for doc in results["documents"][0]:
            context += f"- {doc}\n"
        
        return context
    
    def execute_and_observe(self, task: dict) -> str:
        """
        执行任务并生成 Observation
        """
        # 获取感知上下文
        context = self.perceive_context(task)
        
        # 构建 Prompt（含感知上下文）
        prompt = f"""
        Objective: {self.goal}
        Current Task: {task['description']}
        {context}
        
        Please complete the task and return the result.
        """
        
        # LLM 执行
        result = openai.chat.completions.create(
            model="gpt-4",
            messages=[{"role": "user", "content": prompt}]
        ).choices[0].message.content
        
        # 将结果持久化为新的 Observation
        self._store_observation(task, result)
        
        return result
    
    def _store_observation(self, task: dict, result: str):
        """向量化存储观察结果"""
        embedding = openai.embeddings.create(
            model=self.embedding_model,
            input=result
        ).data[0].embedding
        
        self.collection.add(
            embeddings=[embedding],
            documents=[result],
            metadatas=[{
                "task": task["description"],
                "timestamp": datetime.now().isoformat()
            }],
            ids=[f"obs_{uuid4()}"]
        )
```

**BabyAGI 感知模块的设计启示：**

- **语义检索优于时间排序：** 向量相似度检索确保 Agent 在当前步骤获取"最相关"的历史观察，而非简单的时间倒序。
- **轻量级感知闭环：** 整个感知循环仅 ~150 行代码，展示了感知模块的最小可行实现。
- **多向量数据库支持：** 抽象了 Pinecone、ChromaDB、Qdrant 等多种向量数据库后端，体现了良好的工程抽象。

### 3.2 标准化协议：MCP（Model Context Protocol）

#### 协议定位

**项目地址：** `modelcontextprotocol/specification`（GitHub，由 Anthropic 发起，2025 年 12 月捐赠 Linux 基金会）

#### MCP Server 代码实例

**Python FastMCP 示例 —— 构建一个感知外部 API 的 Server：**

```python
# mcp_perception_server.py
# 基于 FastMCP 构建的感知模块 Server

from fastmcp import FastMCP
import httpx
import json
from typing import Optional

# 创建 MCP Server 实例
mcp = FastMCP("Perception Server")

@mcp.tool()
async def perceive_rest_api(
    url: str,
    method: str = "GET",
    headers: Optional[dict] = None,
    body: Optional[dict] = None
) -> str:
    """
    通用 REST API 感知工具
    
    功能：将任意 REST API 的返回结果采集并转化为结构化 Observation。
    这是感知模块最基础的能力 —— API Response → Observation
    """
    async with httpx.AsyncClient(timeout=30) as client:
        if method.upper() == "GET":
            response = await client.get(url, headers=headers)
        elif method.upper() == "POST":
            response = await client.post(url, json=body, headers=headers)
        else:
            return f"Error: Unsupported method {method}"
    
    # 感知器的核心工作：将原始响应转化为 LLM 友好的 Observation
    return _format_observation(response)

def _format_observation(response: httpx.Response) -> str:
    """
    JSON 响应 → 自然语言 Observation 的转换
    
    这是感知模块中 Parser 角色的核心实现
    """
    status = response.status_code
    size = len(response.content)
    
    observation = f"[API Response] Status: {status}, Size: {size} bytes\n"
    
    try:
        data = response.json()
        # 结构化摘要：提取关键字段而非完整 JSON
        if isinstance(data, dict):
            keys = list(data.keys())[:10]  # 最多展示 10 个顶层 Key
            observation += f"Response keys: {keys}\n"
            # 智能截断：只展示前 500 字符的 JSON
            json_snippet = json.dumps(data, ensure_ascii=False, indent=2)[:500]
            observation += f"Response preview:\n```json\n{json_snippet}\n```"
        elif isinstance(data, list):
            observation += f"Response is a list of {len(data)} items\n"
            if data:
                observation += f"First item preview: {json.dumps(data[0], ensure_ascii=False)[:200]}"
    except json.JSONDecodeError:
        # 非 JSON 响应：按纯文本处理
        text = response.text[:1000]
        observation += f"Response text (first 1000 chars):\n{text}"
    
    return observation

@mcp.tool()
async def perceive_webpage(url: str, extract_mode: str = "text") -> str:
    """
    网页感知工具
    
    功能：抓取网页内容并转化为 LLM 可处理的 Observation。
    支持 'text'（纯文本）和 'html'（保留结构）两种模式。
    """
    from bs4 import BeautifulSoup
    
    async with httpx.AsyncClient() as client:
        response = await client.get(url)
    
    soup = BeautifulSoup(response.text, "html.parser")
    
    if extract_mode == "text":
        # 去除 script/style 标签，提取纯文本
        for tag in soup(["script", "style", "nav", "footer"]):
            tag.decompose()
        text = soup.get_text(separator="\n", strip=True)
        # 压缩空白行
        lines = [line.strip() for line in text.split("\n") if line.strip()]
        text = "\n".join(lines)
        
        # 截断到合理长度（避免撑爆上下文窗口）
        max_chars = 5000
        if len(text) > max_chars:
            text = text[:max_chars] + f"\n...[truncated, {len(text) - max_chars} more chars]"
        
        return f"[Webpage Content from {url}]\n\n{text}"
    
    elif extract_mode == "html":
        # 保留 HTML 结构用于精确解析
        html = soup.prettify()[:10000]
        return f"[HTML Content from {url}]\n\n```html\n{html}\n```"

@mcp.resource("memory://recent_observations")
def get_recent_observations() -> str:
    """

    """
    # 实际实现中从向量数据库或日志系统读取
    return "No recent observations stored."

@mcp.resource("config://perception_rules")
def get_perception_rules() -> str:
    """
    MCP Resource：感知模块的配置规则
    
    功能：告知 Agent 当前感知模块的能力边界和限制。
    这是 MCP 作为 "USB 协议" 的关键特性 —— 
    Agent 可以动态发现 Sensors 的能力。
    """
    return """
    Perception Module Capabilities:
    - REST API monitoring: Supported (GET, POST)
    - Webpage scraping: Supported (text, html modes)
    - File system watching: Supported
    - Image analysis: Requires multimodal model
    - Realtime streaming: Not supported
    - Rate limit: 100 requests/minute
    - Max response size: 50KB

if __name__ == "__main__":
    # 通过 stdio 传输层运行 MCP Server
    mcp.run(transport="stdio")
```

```
│       LangGraph 状态图感知模型           │

│  ┌─────────┐                            │

│  └────┬────┘                            │
│       │                                  │
│       ▼                                  │
│  ┌─────────┐    感知入口                 │
│  │ Agent   │◄───────────────┐           │
```

**最小化 ReAct Agent（含感知模块）代码实现：**

```python
# langgraph_perception_agent.py
# 基于 LangGraph 1.0 API 构建的感知驱动 ReAct Agent

import operator
from typing import Annotated, TypedDict, Sequence
from langchain_core.messages import (
    BaseMessage, HumanMessage, AIMessage, ToolMessage
)
from langgraph.graph import StateGraph, START, END
from langgraph.prebuilt import ToolNode
from langgraph.checkpoint.memory import MemorySaver

# ============================================================
# 1. 定义状态：State 即感知容器
# ============================================================
class AgentState(TypedDict):
    """
    AgentState 是 LangGraph 的核心抽象。
    它同时担任"感知容器"的角色 —— 
    所有 Observations（工具返回、LLM 输出）都累积在此。
    """
    messages: Annotated[Sequence[BaseMessage], operator.add]
    # operator.add 确保新消息追加而非覆盖 → 持续感知累积

# ============================================================
# 2. 定义感知工具
# ============================================================
def search_web(query: str) -> str:
    """
    搜索引擎感知工具
    返回: Observation (自然语言格式)
    """
    # 实际实现：调用 Tavily / Google Search API
    results = tavily_client.search(query, max_results=3)
    observation = f"[Web Search Results for '{query}']:\n"
    for i, r in enumerate(results, 1):
        observation += f"{i}. {r['title']}\n   {r['content'][:200]}\n"
    return observation

def read_file(path: str) -> str:
    """
    文件系统感知工具
    返回: Observation
    """
    with open(path, "r") as f:
        content = f.read(5000)  # 感知截断
    return f"[File Content: {path}]\n```\n{content}\n```"

def run_sql(query: str) -> str:
    """
    数据库感知工具
    返回: Observation
    """
    import sqlite3
    conn = sqlite3.connect("agent_memory.db")
    cursor = conn.execute(query)
    rows = cursor.fetchall()
    # 将数据库结果格式化为自然语言 Observation
    if not rows:
        return "[SQL Result] Query returned 0 rows."
    columns = [desc[0] for desc in cursor.description]
    observation = f"[SQL Result] {len(rows)} rows returned.\n"
    observation += "Columns: " + ", ".join(columns) + "\n"
    observation += f"First 5 rows:\n{rows[:5]}"
    return observation

# ============================================================
# 3. 构建状态图（感知闭环）
# ============================================================
tools = [search_web, read_file, run_sql]

# 将工具绑定到 LLM
llm_with_tools = chat_model.bind_tools(tools)

# 定义 Agent 节点（思考节点）
def agent_node(state: AgentState) -> dict:
    """
    思考节点：接收当前所有感知结果（State.messages），
    进行推理并决定下一步行动。
    
    state.messages 中包含了：
    - 用户的原始输入
    - 历史上所有工具执行的 Observation
    - LLM 之前的推理
    """
    response = llm_with_tools.invoke(state["messages"])
    return {"messages": [response]}

# 定义路由逻辑
def should_continue(state: AgentState) -> str:
    """
    路由条件：基于 LLM 的输出决定是否继续感知
    """
    last_message = state["messages"][-1]
    if hasattr(last_message, "tool_calls") and last_message.tool_calls:
        return "tools"  # 继续感知循环
    return END  # 结束

# 构建图
graph = StateGraph(AgentState)

# 添加节点
graph.add_node("agent", agent_node)           # 思考节点
graph.add_node("tools", ToolNode(tools))       # 感知/执行节点

# 添加边
graph.add_edge(START, "agent")
graph.add_conditional_edges(
    "agent",
    should_continue,
    {"tools": "tools", END: END}
)
graph.add_edge("tools", "agent")  # 工具 Observations 回流到思考节点

# 编译（带持久化记忆）
memory = MemorySaver()  # 基于 SQLite 的检查点持久化
agent = graph.compile(checkpointer=memory)

# ============================================================
# 4. 运行：观察感知闭环
# ============================================================
config = {"configurable": {"thread_id": "session-001"}}

# 每一轮 messages 中都会累积感知结果
for event in agent.stream(
    {"messages": [HumanMessage(content="分析最近的销售数据趋势")]},
    config=config
):
    # event 中的 ToolMessage 即感知 Observation
    for node_name, node_output in event.items():
        if node_name == "tools":
            print(f"[感知] 工具返回 Observation: {node_output}")
        elif node_name == "agent":
            print(f"[思考] LLM 推理: {node_output}")
```

**LangGraph 感知模式的设计精华：**

| 设计要素 | 实现方式 | 感知意义 |
|---------|---------|---------|
| **State 累积** | `operator.add` 注解 | Observations 永不丢失，全程可追溯 |
| **Conditional Edges** | `should_continue()` | 动态决策是否继续感知，避免无效循环 |
| **MemorySaver** | SQLite Checkpoint | 感知历史跨会话持久化 |
| **ToolNode** | 预构建组件 | 工具调用结果自动转为 `ToolMessage` Observation |
| **流式输出** | `.stream()` | 实时观察每一步的感知过程 |

### 4.1 解析器模式：JSON → 自然语言 Observation

**Parser（解析器）** 是感知系统中连接"机器世界"与"LLM 世界"的关键桥梁。其核心任务是将结构化/半结构化的机器数据转化为 LLM 可精确理解的**自然语言 Observation**。

#### 设计原则

```
┌─────────────────────────────────────────────┐
│            Parser 设计模式                   │
│                                              │
│  输入：Machine-Readable Data                 │
│  ┌──────────────────────────────┐           │
│  │ {"users": [                  │           │
│  │   {"id":1, "name":"Alice",   │           │
│  │    "orders":143,             │           │
│  │    "revenue":45280.50}       │           │
│  │ ]}                           │           │
│  └──────────────┬───────────────┘           │
│                 │                            │
│                 ▼                            │
│  ┌──────────────────────────────┐           │
│  │        Parser Pipeline       │           │
│  │                              │           │
│  │  1. 结构识别 (Schema Detect) │           │
│  │  2. 关键字段提取 (Key Extract)│           │
│  │  3. 信息摘要 (Summarize)     │           │
│  │  4. 格式化输出 (Format)      │           │
│  └──────────────┬───────────────┘           │
│                 │                            │
│                 ▼                            │
│  输出：LLM-Friendly Observation              │
│  ┌──────────────────────────────┐           │
│  │ [API Observation]            │           │
│  │ Query: /api/users            │           │
│  │ Returned: 1 user record      │           │
│  │ Key Data:                    │           │
│  │ • Alice: 143 orders,         │           │
│  │   $45,280.50 total revenue   │           │
│  └──────────────────────────────┘           │
└─────────────────────────────────────────────┘
```

**核心设计原则：**

1. **保真度优先（Fidelity First）：** 不丢失关键信息；数值精度必须保留。
2. **结构保留（Structure Preservation）：** 层次关系用缩进/Markdown 表格表达，而非扁平化。
3. **智能截断（Smart Truncation）：** 超越 Token 上限时，优先保留 Head + 统计摘要，而非简单 Tail-cut。
4. **元数据注入（Metadata Injection）：** 每条 Observation 应包含时间戳、来源、数据类型标签。

#### 通用 Parser 实现

```python
# observation_parser.py
# 通用感知解析器 —— 将各类数据格式转化为标准 Observation

import json
from typing import Any, Optional
from datetime import datetime
from dataclasses import dataclass, field

@dataclass
class Observation:
    """标准 Observation 数据结构"""
    content: str                          # 自然语言内容
    source: str                           # 来源标识 (API/File/Web/DB)
    timestamp: str = field(default_factory=lambda: datetime.now().isoformat())
    metadata: dict = field(default_factory=dict)
    token_estimate: int = 0

class ObservationParser:
    """
    通用感知解析器
    
    支持格式：JSON / HTML / CSV / SQL Result / Plain Text
    输出格式：统一 Observation
    """
    
    MAX_TOKENS = 4000  # 默认 Observation 最大 Token 数
    
    def parse(
        self, 
        raw_data: Any, 
        source: str = "unknown",
        format_hint: Optional[str] = None
    ) -> Observation:
        """
        主解析入口：自动检测格式并解析
        """
        if format_hint == "json" or isinstance(raw_data, (dict, list)):
            return self._parse_json(raw_data, source)
        elif format_hint == "html" or (
            isinstance(raw_data, str) and raw_data.strip().startswith("<")
        ):
            return self._parse_html(raw_data, source)
        elif format_hint == "csv":
            return self._parse_csv(raw_data, source)
        else:
            return self._parse_text(str(raw_data), source)
    
    def _parse_json(self, data: dict | list, source: str) -> Observation:
        """
        JSON 解析策略：
        1. 提取顶层结构信息（Keys、长度）
        2. 采样关键字段
        3. 格式化为结构化文本
        """
        content_parts = [f"[Observation from {source}]"]
        
        if isinstance(data, dict):
            keys = list(data.keys())
            content_parts.append(f"Type: Object with {len(keys)} keys: {keys[:15]}")
            
            # 智能采样：小对象全量展示，大对象提取摘要
            json_str = json.dumps(data, ensure_ascii=False, indent=2)
            if len(json_str) <= 2000:
                content_parts.append(f"Full Data:\n```json\n{json_str}\n```")
            else:
                # 大对象：提取关键字段摘要
                content_parts.append("Key Data (sampled):")
                for key in keys[:10]:  # 最多 10 个顶层字段
                    val = data[key]
                    if isinstance(val, (int, float, str, bool)):
                        content_parts.append(f"  • {key}: {val}")
                    elif isinstance(val, list):
                        content_parts.append(f"  • {key}: list[{len(val)}]")
                    elif isinstance(val, dict):
                        content_parts.append(f"  • {key}: dict[{len(val)} keys]")
                
                # 追加尾部截断的全量数据
                truncated = json_str[:self.MAX_TOKENS * 4]
                content_parts.append(f"\nRaw JSON (truncated):\n```json\n{truncated}\n```")
        
        elif isinstance(data, list):
            content_parts.append(f"Type: Array with {len(data)} items")
            if data and len(data) <= 5:
                for i, item in enumerate(data):
                    if isinstance(item, dict):
                        content_parts.append(
                            f"  [{i}] {json.dumps(item, ensure_ascii=False)[:200]}"
                        )
            elif data:
                content_parts.append(f"  First item: {json.dumps(data[0], ensure_ascii=False)[:300]}")
                content_parts.append(f"  Last item: {json.dumps(data[-1], ensure_ascii=False)[:300]}")
        
        content = "\n".join(content_parts)
        return Observation(
            content=content,
            source=source,
            metadata={"format": "json", "size": len(json.dumps(data, ensure_ascii=False))}
        )
    
    def _parse_html(self, html: str, source: str) -> Observation:
        """
        HTML 解析策略：
        1. 提取 title/meta
        2. 提取关键文本内容
        3. 保留链接结构
        """
        from bs4 import BeautifulSoup
        
        soup = BeautifulSoup(html, "html.parser")
        
        content_parts = [f"[HTML Observation from {source}]"]
        
        # 提取标题
        title = soup.title.string if soup.title else "No title"
        content_parts.append(f"Page Title: {title}")
        
        # 提取 Meta 描述
        meta_desc = soup.find("meta", attrs={"name": "description"})
        if meta_desc:
            content_parts.append(f"Meta Description: {meta_desc.get('content', '')}")
        
        # 提取链接结构
        links = soup.find_all("a", href=True)[:20]
        if links:
            content_parts.append("\nLinks found:")
            for link in links:
                content_parts.append(f"  • {link.get_text(strip=True)[:50]} → {link['href'][:80]}")
        
        # 提取文本主体
        for tag in soup(["script", "style", "nav", "footer", "header"]):
            tag.decompose()
        
        body_text = soup.get_text(separator="\n", strip=True)
        lines = [l.strip() for l in body_text.split("\n") if l.strip()]
        body_text = "\n".join(lines)
        
        # 截断
        if len(body_text) > self.MAX_TOKENS * 4:
            body_text = body_text[:self.MAX_TOKENS * 4] + f"\n...[truncated]"
        
        content_parts.append(f"\nPage Content:\n{body_text}")
        
        return Observation(
            content="\n".join(content_parts),
            source=source,
            metadata={"format": "html", "title": title}
        )
    
    def _parse_text(self, text: str, source: str) -> Observation:
        """纯文本解析：直接截断包装"""
        if len(text) > self.MAX_TOKENS * 4:
            text = text[:self.MAX_TOKENS * 4] + f"\n...[truncated, original length: {len(text)} chars]"
        
        return Observation(
            content=f"[Text Observation from {source}]\n\n{text}",
            source=source,
            metadata={"format": "text", "length": len(text)}
        )
    
    def _parse_csv(self, csv_text: str, source: str) -> Observation:
        """CSV 解析：提取表头和前 N 行"""
        import csv
        from io import StringIO
        
        reader = csv.reader(StringIO(csv_text))
        rows = list(reader)
        
        content = f"[CSV Observation from {source}]\n"
        content += f"Total rows: {len(rows)}\n"
        
        if rows:
            content += f"Headers: {rows[0]}\n"
            content += f"Data preview (first 10 rows):\n"
            for row in rows[1:11]:
                content += f"  {row}\n"
        
        return Observation(
            content=content,
            source=source,
            metadata={"format": "csv", "rows": len(rows)}
        )
```

### 4.2 可观测性在感知模块中的应用

**可观测性（Observability）** 是感知模块走向生产化的必要条件。一个不具备可观测性的感知系统，就像黑箱内的传感器——我们无法知道它看到了什么、漏掉了什么、误解了什么。

#### Trace-First 架构

```
┌──────────────────────────────────────────────────┐
│           Trace-First 感知可观测架构              │
│                                                   │
│  ┌─────────────────────────────────────────┐    │
│  │         Agent Execution                  │    │
│  │                                          │    │
│  │  Span: "Agent.Run"                       │    │
│  │  ├── Span: "Perception.Init"            │    │
│  │  │   ├── Event: "context_assembled"     │    │
│  │  │   └── Metric: "context_tokens"=1500  │    │
│  │  ├── Span: "LLM.Think"                  │    │
│  │  │   ├── Event: "prompt_sent"           │    │
│  │  │   └── Event: "response_received"     │    │
│  │  ├── Span: "Action.Execute"             │    │
```

#### 基于 OpenTelemetry 的感知追踪实现

```python
# perception_observability.py
# 基于 OpenTelemetry 的感知模块可观测性实现

from opentelemetry import trace, metrics
from opentelemetry.sdk.trace import TracerProvider
from opentelemetry.sdk.metrics import MeterProvider
from opentelemetry.exporter.otlp.proto.grpc.trace_exporter import OTLPSpanExporter
from opentelemetry.exporter.otlp.proto.grpc.metric_exporter import OTLPMetricExporter
from opentelemetry.sdk.resources import SERVICE_NAME, Resource
from opentelemetry.instrumentation.langchain import LangChainInstrumentor
from functools import wraps
import time

# ============================================================
# 1. 初始化 OpenTelemetry
# ============================================================
resource = Resource(attributes={SERVICE_NAME: "agent-perception-module"})

# Tracer
tracer_provider = TracerProvider(resource=resource)
tracer_provider.add_span_processor(
    trace.export.BatchSpanProcessor(OTLPSpanExporter())
)
trace.set_tracer_provider(tracer_provider)
tracer = trace.get_tracer(__name__)

# Meter
meter_provider = MeterProvider(resource=resource, metric_readers=[])
metrics.set_meter_provider(meter_provider)
meter = metrics.get_meter(__name__)

# ============================================================
# 2. 定义感知度量指标
# ============================================================
# 感知延迟直方图
perception_latency = meter.create_histogram(
    name="perception.latency",
    description="Perception operation latency in milliseconds",
    unit="ms"
)

# 感知数据量计数器
perception_data_size = meter.create_counter(
    name="perception.data_size",
    description="Total bytes processed by perception module",
    unit="bytes"
)

# Observation Token 数直方图
observation_tokens = meter.create_histogram(
    name="perception.observation_tokens",
    description="Token count of generated observations",
    unit="tokens"
)

# ============================================================
# 3. 可观测装饰器
# ============================================================
def trace_perception(operation: str):
    """
    为感知操作添加分布式追踪
    
    每一次感知操作都会生成一个 Span，
    记录其输入、输出、耗时、状态等关键信息。
    """
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            with tracer.start_as_current_span(
                f"Perception.{operation}",
                attributes={
                    "perception.operation": operation,
                    "perception.source": kwargs.get("source", "unknown"),
                }
            ) as span:
                start_time = time.time()
                
                try:
                    # 记录输入信息
                    span.set_attribute(
                        "perception.input_size",
                        len(str(kwargs.get("raw_data", "")))
                    )
                    
                    # 执行感知操作
                    result = func(*args, **kwargs)
                    
                    # 记录输出指标
                    elapsed_ms = (time.time() - start_time) * 1000
                    perception_latency.record(elapsed_ms, {
                        "operation": operation,
                        "status": "success"
                    })
                    
                    if hasattr(result, 'content'):
                        obs_tokens = len(result.content) // 4  # 粗略 Token 估算
                        observation_tokens.record(obs_tokens, {
                            "operation": operation
                        })
                    
                    span.set_attribute("perception.latency_ms", elapsed_ms)
                    span.set_attribute("perception.status", "success")
                    
                    return result
                    
                except Exception as e:
                    elapsed_ms = (time.time() - start_time) * 1000
                    perception_latency.record(elapsed_ms, {
                        "operation": operation,
                        "status": "error"
                    })
                    span.set_attribute("perception.status", "error")
                    span.set_attribute("perception.error", str(e))
                    span.record_exception(e)
                    raise
                    
        return wrapper
    return decorator

# ============================================================
# 4. 应用追踪到感知模块
# ============================================================
class TraceablePerceptionModule:
    """
    可追踪的感知模块
    
    每个感知操作都会：
    1. 生成 OpenTelemetry Span
    2. 记录关键指标（延迟、数据量、Token 数）
    3. 关联到上级 Agent Span
    """
    
    def __init__(self, parser: ObservationParser):
        self.parser = parser
    
    @trace_perception("api_call")
    def perceive_api(
        self, url: str, method: str = "GET", source: str = "rest_api"
    ) -> Observation:
        """带追踪的 API 感知"""
        import httpx
        
        with tracer.start_as_current_span("HTTP.Request") as span:
            span.set_attribute("http.url", url)
            span.set_attribute("http.method", method)
            
            response = httpx.get(url, timeout=30)
            
            span.set_attribute("http.status_code", response.status_code)
            span.set_attribute("http.response_size", len(response.content))
            
            # 记录数据量
            perception_data_size.add(
                len(response.content),
                {"source": source, "format": "json"}
            )
        
        return self.parser.parse(response.json(), source=source, format_hint="json")
    
    @trace_perception("file_read")
    def perceive_file(self, path: str) -> Observation:
        """带追踪的文件感知"""
        with open(path, "r") as f:
            content = f.read()
        
        perception_data_size.add(
            len(content),
            {"source": "filesystem", "format": path.split(".")[-1]}
        )
        
        return self.parser.parse(content, source=f"file://{path}")
    
    @trace_perception("web_scrape")
    def perceive_webpage(self, url: str) -> Observation:
        """带追踪的网页感知"""
        import httpx
        
        response = httpx.get(url, timeout=30)
        html = response.text
        
        perception_data_size.add(
            len(html),
            {"source": "web", "format": "html"}
        )
        
        return self.parser.parse(html, source=f"web://{url}", format_hint="html")

# ============================================================
# 5. 集成 LangChain/LangGraph 追踪
# ============================================================
# 自动为 LangChain/LangGraph 的所有操作注入追踪
LangChainInstrumentor().instrument()

# LangSmith 集成（LangChain 官方可观测平台）
import os
os.environ["LANGCHAIN_TRACING_V2"] = "true"
os.environ["LANGCHAIN_ENDPOINT"] = "https://api.smith.langchain.com"
os.environ["LANGCHAIN_API_KEY"] = "ls__your_key"
os.environ["LANGCHAIN_PROJECT"] = "agent-perception"
```

**Trace-First 架构的关键价值：**

| 维度 | 传统日志 | Trace-First |
|------|---------|-------------|
| **可视化** | 分散的文本日志 | 完整的 Span 瀑布图 |
| **关联性** | 手动 grep/correlation_id | 自动父子 Span 关联 |
| **性能分析** | 手动计时 | 自动延迟直方图 + P50/P95/P99 |
| **错误定位** | 需要逐行排查 | 异常 Span 直接高亮 |
| **感知洞察** | 无法分析 | 每条 Observation 的来源、大小、延迟一目了然 |

## 五、总结与展望

### 核心结论

1. **感知模块是 Agent 的"感官系统"，其成熟度决定了 Agent 的上限。** 从 PEAS 的 Sensors 到现代 Agent Loop 的 Observe 环节，感知模块已从"Stringify Everything"进化为**全域思考（All-field Thinking）**。

2. **MCP 协议正在重塑感知模块的集成范式。** `N×M → N+M` 的复杂度降低使感知能力的生态化扩展成为可能，13,000+ 公开 MCP Server 证明了这一趋势的不可逆转。

3. **开源框架为感知工程化提供了坚实基座。** LangGraph 的状态图、OpenClaw 的观察者模式、Qwen-Agent 的端到端多模态，各自解决感知模块的不同痛点。

4. **可观测性是感知模块生产化的"最后一块拼图"。** Trace-first 架构让每一次感知操作可追溯、可度量、可优化。

### 趋势展望

```
┌─────────────────────────────────────────────┐
│         感知模块的技术演进方向                │
│                                              │
│  2024 ────→ 2025 ────→ 2026 ────→ 2027      │
│                                              │
│  纯文本      MCP标准化    全域思考      ???   │
│  上下文注入   + 向量记忆    多模态融合    感知即推理│
│                                              │
│  单Agent     工具链       Agent Swarm   群体感知│
│  串行感知    可组装        并行感知      涌现感知│
└─────────────────────────────────────────────┘
```

**未来方向：**

- **感知即推理（Perception as Reasoning）：** 感知不再是简单的数据采集，而是与推理深度融合——Agent 在感知的同时进行假设验证。
- **群体感知（Swarm Perception）：** 多个 Agent 以不同视角感知同一环境，交叉验证、相互校准。
- **自适应感知粒度：** 根据任务复杂度动态调整感知深度（粗略扫描 vs 精细分析），避免信息过载和 Token 浪费。
- **感知安全（Perception Security）：** 防止恶意信号注入（Prompt Injection via Tool Output）、感知结果投毒等攻击向量。
